## 五、技术实现：H5付费门控

### 5.1 最简方案（无需后端）

```javascript
// 在index.html中添加付费门控
const PAID_TOKENS = {
  'free2025': { expire: null },           // 免费体验码
  'vip_abc123': { expire: '2026-07-10' }, // 付费token
};

function checkAccess() {
  const urlParams = new URLSearchParams(window.location.search);
  const token = urlParams.get('token');
  
  if (!token) {
    showPayWall(); // 显示付费墙
    return;
  }
  
  const info = PAID_TOKENS[token];
  if (!info) {
    showPayWall();
    return;
  }
  
  if (info.expire && new Date(info.expire) < new Date()) {
    showPayWall();
    return;
  }
  
  // 通过验证，正常使用
  localStorage.setItem('access_token', token);
  localStorage.setItem('access_time', Date.now());
}
```

### 5.2 进阶方案（简易后端）

用Cloudflare Workers或Vercel Serverless做token验证：
1. 用户付款 → 生成唯一token存入KV
2. 用户打开链接 → Worker验证token
3. 24小时后自动过期

### 5.3 推荐起步方案

**阶段1（0-100单）**：手动生成token，写在代码里，通过私信发送
**阶段2（100-500单）**：用Google Sheets做简易后端（免费）
**阶段3（500+单）**：升级到Cloudflare Workers + KV

## 八、风险与应对

| 风险 | 应对 |
|------|------|
| 小红书限流外部链接 | 用图片+评论区引导私信，不发裸链 |
| 闲鱼虚拟商品被下架 | 用「卡密兑换」模式，实物发货 |
| DeepSeek API成本上升 | 缓存常见问题，限制单用户调用次数 |
| 竞品模仿 | 建立品牌壁垒：人设IP化+用户数据积累 |
| 用户退款纠纷 | 设置24小时无理由退款，降低差评风险 |
